Google Calendar support is planned for release campaigns, content drops,
fan emails, and tour-date reminders. The goal is to let Aria propose
calendar events from release plans while still requiring your approval
before anything is written.

<TierBadge tier="all" /> &nbsp; Coming soon. Calendar connection is not a public self-serve integration yet.

## What Aria can do

| Action | Default behavior |
|---|---|
| Read your calendar | Planned |
| Propose new events around existing ones | Planned |
| Add a single event | Planned, with confirmation |
| Add a release-plan task bundle | Planned, with batch confirmation |
| Move / reschedule events | Planned, with confirmation |
| Delete events | Not planned for v1 |
| Read attendee responses | Planned for scheduling workflows |

## Connecting

<Steps>
  <Step title="Settings → Connected Platforms → Google Calendar">
    Calendar is not self-serve yet. When it ships, it will appear in
    Settings → Connected Platforms.
  </Step>
  <Step title="Approve scopes">
    Patchline will request the minimum calendar scopes needed for the
    actions you enable.
  </Step>
  <Step title="Confirm">
    The connection will show in Connected Platforms when OAuth completes.
  </Step>
</Steps>

## Privacy

- OAuth tokens stored encrypted, per-user.
- Calendar events will be read only when you ask.
- New events are created only with your explicit confirmation.

## FAQ

<AccordionGroup>
  <Accordion title="Will Aria spam my calendar?">
    No. Aria proposes; you approve. Batch operations (like a 28-event
    release plan) get one batch-approval prompt, not 28 individual
    prompts.
  </Accordion>
  <Accordion title="Can I use Outlook / Apple Calendar?">
    Google Calendar is the first planned calendar integration. Outlook
    is on the roadmap after that. Apple Calendar via CalDAV is not
    currently planned.
  </Accordion>
  <Accordion title="What calendar does Aria use if I have multiple?">
    Your primary calendar by default. Multi-calendar support
    (assign release-plan events to a specific calendar) is on the
    roadmap.
  </Accordion>
</AccordionGroup>
